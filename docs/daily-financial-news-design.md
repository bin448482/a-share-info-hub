# 当日财经信息总结 Skill 设计

本文档定义“当日财经信息总结”第一版设计。根据 `daily-financial-news-feasibility-analysis.md` 的结论，第一版不再建设爬虫、标准化表、DuckDB 或每日新闻采集管线，而是制作一个 repo 内 skill：由 LLM 围绕 `US Macro` 和 `Investment Bank Views` 等主题主动读取公开信息、理解上下文、生成当日摘要，并在结论中标明参考信息源。

## 背景和目标

原设计尝试把当日财经信息做成可采集的数据包。但可行性分析显示：

- `US Macro` 数据源可通过 AKShare 等结构化接口获取，但这更像数据事实层，不需要重新做新闻爬虫。
- `Investment Bank Views` 没有稳定、免费、结构化的直接 API，公开内容主要来自媒体转述、研报摘要或页面信息。
- 若为了投行观点做页面抓取，会引入登录态、反爬、结构漂移和低密度内容问题。

因此第一版目标调整为：

- 建立一个 LLM 使用的 skill，而不是新增数据爬虫。
- 让 LLM 围绕指定主题读取、筛选、理解公开信息。
- 输出当日财经信息总结、潜在 A 股影响主题、风险观察和待验证问题。
- 每个关键结论必须引用参考来源，说明信息来自哪里。
- 不提供交易建议，不把外部信息直接写成 A 股确定性结论。

## 第一版范围

第一版 skill 聚焦两类主题：

1. `US Macro`
   - 美国 CPI、核心 CPI、PCE、非农、失业率、FOMC、利率预期、美债收益率、美元流动性代理指标。
   - 重点解释“事实是什么、市场预期是什么、变化方向是什么、可能影响哪些 A 股研究主题”。

2. `Investment Bank Views`
   - 高盛、摩根士丹利、摩根大通、花旗、瑞银等主要投行的公开后市判断。
   - 只使用公开可访问的信息、媒体转述或研报摘要，不要求覆盖每家投行，也不要求每日都有结果。

明确不做：

- 不新增 `news-update` CLI。
- 不建设新闻爬虫。
- 不新增 `daily_financial_news.parquet`。
- 不写入 DuckDB。
- 不抓取自选博主或微信大 V。
- 不聚合华尔街和主流财经媒体每日前 5 条。
- 不绕过登录、验证码、订阅墙或反爬限制。
- 不输出买卖、仓位、目标价、止盈止损等交易行动建议。

## Skill 形态

建议新增 repo 内 skill：

```text
skills/
  daily-financial-briefing/
    AGENTS.md
    claude.md
    SKILL.md
    references/
      AGENTS.md
      claude.md
      source-routing.md
      output-schema.md
      citation-rules.md
```

如果同时需要 Claude Code 使用，应按仓库既有模式镜像到 `.claude/skills/daily-financial-briefing/`。

`SKILL.md` 保持精简，只描述触发条件、工作流和必须遵守的边界。详细主题定义、来源路由、输出格式和引用规则放入 `references/`。

## 使用方式

用户可以用自然语言触发：

```text
使用 $daily-financial-briefing，总结今天 US Macro 和主要投行观点对 A 股研究的影响。
```

也可以指定日期和关注点：

```text
使用 $daily-financial-briefing，日期 2026-06-19，重点看美国利率预期、CPI、非农和高盛/摩根对中国资产的观点。
```

Skill 输出不直接改写每日行情数据，也不要求先运行 `daily-update`。如果用户希望接入每日复盘，输出应作为 `daily-review` 的外部背景材料，由复盘流程再结合 A 股行情证据使用。

## 工作流

Skill 执行时按以下顺序工作：

1. 明确日期和范围
   - 默认使用用户指定日期。
   - 未指定日期时使用当前日期。
   - 明确本次覆盖 `US Macro`、`Investment Bank Views`，以及用户额外指定的主题。

2. 收集公开信息
   - 优先读取权威或原始来源。
   - 对宏观数据，优先使用官方发布机构、交易所/利率工具、AKShare 可验证接口或可靠财经数据转述。
   - 对投行观点，优先使用投行公开文章、公开报告摘要、主流财经媒体转述或可访问研报摘要。
   - 不使用无法访问、需要登录或无法引用来源的内容。

3. 归类和去重
   - 将信息归入 `US Macro`、`Investment Bank Views` 或用户指定主题。
   - 多个来源重复描述同一事件时合并为一个事实，不重复堆叠。
   - 投行观点与宏观事实分开，不把观点写成事实。

4. 形成结论
   - 每条结论必须说明依据来自哪些来源。
   - 区分事实、市场预期、机构观点、推论和待验证问题。
   - 对 A 股影响只写成研究假设或风险观察，不写成确定性方向。

5. 输出总结
   - 输出可读的当日财经简报。
   - 附带引用来源列表。
   - 标注信息缺口和低可信度部分。

## 来源路由

第一版不固定抓取源清单，而是定义来源优先级。

`US Macro` 优先级：

1. 官方或准官方来源，例如 BLS、BEA、Federal Reserve、CME FedWatch、美国财政部等。
2. 可验证数据接口或本地工具，例如 AKShare 已验证宏观接口、美债收益率接口。
3. 主流财经媒体对官方数据的转述。

`Investment Bank Views` 优先级：

1. 投行公开发布页面或公开报告摘要。
2. 主流财经媒体对投行观点的转述。
3. 可访问的研报摘要或财经资讯聚合。

禁止来源：

- 登录后内容。
- 付费墙后内容。
- 无法给出 URL 或清晰出处的内容。
- 用户未要求的自媒体、微信大 V、社媒帖子。
- 来源不明的二次转载。

## 输出格式

建议输出 Markdown：

```text
# 当日财经信息简报

- 日期：YYYY-MM-DD
- 范围：US Macro / Investment Bank Views
- 结论性质：研究背景，不构成投资建议

## 1. 核心结论

1. [结论]
   - 依据：[来源名称，发布时间或访问日期，URL]
   - 类型：事实 / 市场预期 / 投行观点 / 推论
   - A 股研究含义：[候选主题或风险观察]

## 2. US Macro

...

## 3. Investment Bank Views

...

## 4. 对 A 股研究的待验证问题

...

## 5. 信息缺口和边界

...

## 6. 参考来源

- [来源名称] 标题，发布时间或访问日期，URL
```

## 引用规则

- 每条核心结论至少引用一个来源。
- 如果一个结论结合了宏观事实和投行观点，应分别引用两类来源。
- 引用必须包含来源名称和 URL；能取得发布时间时必须包含发布时间。
- 不能引用“我看到有报道”“市场消息称”这类无出处表述。
- 对无法核验的内容，只能放入“信息缺口和边界”，不能作为核心结论。
- 不能过度摘录原文；引用用于定位来源，正文以摘要和转述为主。

## 与每日复盘的关系

该 skill 的输出是外部背景材料，不是行情证据本身。

可以进入每日复盘的内容：

- 外部宏观变化触发的候选主题。
- 投行观点带来的风险观察。
- 需要 A 股行情、板块、情绪或事件数据确认的问题。

不能进入每日复盘的内容：

- 未引用来源的结论。
- 单条外部新闻直接推导出的 A 股确定性判断。
- 投行观点直接变成投资建议。
- 缺少 A 股内部证据确认的个股或板块强结论。

## 异常和降级

- 如果当天没有可用投行观点，输出“未找到可引用的当日公开投行观点”，不编造内容。
- 如果宏观数据尚未发布，只能总结市场预期、发布时间和待观察指标。
- 如果多个来源说法冲突，保留冲突并说明各自来源。
- 如果信息源只能访问标题，不能把标题扩展成未验证的正文结论。
- 如果无法联网或无法读取外部信息，skill 应停止在 `blocked` 边界，并说明缺少可引用来源。

## 测试和验收

第一版完成后需要验证：

- Skill 能被自然语言请求触发。
- 输出包含 `US Macro` 和 `Investment Bank Views` 两个主题区。
- 核心结论都带来源引用。
- 无可用投行观点时不会编造。
- 输出不会包含买卖、仓位、目标价、止盈止损等交易行动建议。
- 输出能明确区分事实、观点、推论和待验证问题。
- 若接入每日复盘，只作为外部背景材料，不改变行情数据契约。

## 后续可选扩展

以下内容不进入第一版：

- 自动采集和落库。
- 固定 source registry。
- 页面抓取脚本。
- 每日定时任务。
- 结构化评分模型。
- 自选博主、微信大 V 和社媒来源。
- 主流媒体前 5 条新闻聚合。
